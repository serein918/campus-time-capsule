# 校园信息封存与分享平台（Campus Time Capsule Platform）

## 📖 项目简介

校园信息封存与分享平台是一个基于前后端分离架构开发的 Web 应用，旨在为大学生提供一个记录目标、梦想、心情和珍贵回忆的平台。

用户可以创建属于自己的"时光胶囊"，设置分类、公开范围以及未来的解封时间，在指定时间到达后才能查看内容，也可以选择公开分享至广场，与其他同学进行互动交流。

系统采用前后端分离开发模式，包含普通用户端和后台管理端两部分，实现了用户管理、时光胶囊管理、点赞收藏、评论互动、公告管理、数据统计等功能，并集成百度智能云文本审核接口，对用户发布内容进行智能审核。

# 💻 本地开发

## 1. 初始化数据库

```bash
mysql -u root -p < database/capsule_db.sql
```

## 2. 启动后端

```bash
cd capsule-server
./mvnw spring-boot:run
```

默认访问地址：

```
http://localhost:8080
```

## 3. 启动前端

```bash
cd capsule-web

npm install

npm run dev
```

默认访问地址：

```
http://localhost:5173
```

# ⚡ 核心实现

## 时光胶囊自动解封

项目最核心的业务逻辑是"时光胶囊解封"。

开发初期曾考虑采用定时任务扫描数据库，定期修改胶囊状态，但这种方案会增加数据库更新压力，也会占用服务器资源。

最终采用**查询时动态判断**的实现方式，在公开广场查询时直接加入：

```sql
open_time <= NOW()
```

数据库仅返回已经达到解封时间的数据，无需后台定时更新状态。

该方案具有以下优点：

- 无需定时任务
- 减少数据库写操作
- 降低服务器资源消耗
- 提高系统可维护性
- 更适合数据量较大的场景
